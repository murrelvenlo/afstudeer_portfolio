Stageopdracht

Achtergrondinformatie

Als stagiair bij Netropolix, had ik als opdracht om aan een webapplicatie te werken. Deze applicatie is bedoeld om bedrijven de mogelijkheid te geven elkaar te nomineren en alzo goede doelen te sponsoren.

Als voorbereiding heb ik een usecase gemaakt. Aan de hand van de usecasebeschrijving heb ik rekening gehouden met de verwachtingen van de klant. Naar aanleiding van de functionaliteiten, heb ik een ERD-model kunnen samenstellen. Dit model bestaat uit 4 tabellen:

  • charity
  • Donation
  • User
  • Social platform

Om verder te kunnen heb ik in "Microsoft Sequal Server Management" het datamodel vertaald naar de Database.

De API is gemaakt in .NET Core Web API. Dit onderdeel van C# had ik nog niet gebruikt, maar met mijn kennis van .NET framework was het vrij makkelijk om de CRUD-operaties toe te passen bij het uitschrijven van de API.

Zodra de API gebruiksklaar was, ben ik dan beginnen werken aan de webapplicatie. De applicatie werd gemaakt met de Javascript framework, Vue.js. Ik kon vrij werken met deze nieuwe framework, omdat het vergelijkbaar is met React.js.

Tijdens mijn stage bij Netropolix heb ik nieuwe IT-programma’s en vaardigheden aangeleerd en toegepast.

Ik heb kennis gemaakt met nieuwe programma’s uit de IT-wereld en heb geleerd om ze te gebruiken.

Een bedrijf zoals Netropolix gebruikt ook haar eigen geschreven en aangepaste software of andere programma’s. Deze software en programma’s zijn meestal specifiek gericht op de service die het bedrijf levert aan de klanten. Ik vond het verfrissend en boeiend om nieuwe items onder de knie te krijgen. Zoals:

  • Admind.WEB.Software (een software pakket voor online administratie)
  • Programmeren in Vue.js
  • Introductie van Vuetify
  • Entity Framework Core
  • Een API maken in .NET Core
Het blijft een uitdaging om in IT te werken omdat je steeds nieuwe dingen mag en kan bijleren.

Ik heb ook de kans gekregen om ook andere dan specifieke IT- vaardigheden aan te leren en te gebruiken.

Het opstellen van ‘een Plan van Aanpak’

Ik heb geleerd dat door het opstellen van ‘een plan van aanpak’ het mezelf makkelijker maakt om de opgelegde doelen te bereiken. Het geeft ook mijn collega’s inzicht in zowel mijn werkmethode als de tijdslijn die ik heb uitgezet. Ik ervaarde wel dat ‘een plan van aanpak’ soms ook wel moet worden bijgestuurd en aangepast omwille van onverwachte voorvallen zoals dingen die meer tijd vragen of technische veranderingen.

Klantgericht werken

De feitelijke opdrachtgever van het project dat ik moest uitwerken was het marketingteam van Netropolix. Ik heb ervaren dat het belangrijk is om af te toetsen wat de concrete wensen zijn van de klant, met hen te overleggen en te evalueren en hun vraag zo goed mogelijk te realiseren om hun verwachtingen in te lossen. Bij de uitwerking van mijn project dacht ‘de klant’ meestal vanuit een visuele voorstelling, zij staan niet altijd stil bij de realiseerbaarheid en de moeilijkheden bij de uitwerking ervan. Dit was een goede leerschool voor mezelf: ‘klant is koning’ en ik als IT-er ben diegene die de juiste skills moet aanwenden om hun wensen te realiseren. Ik merk dat ik bij de uiteindelijke realisatie ook wel veel voldoening beleefde om te zien dat het uitgewerkte project enthousiast ontvangen werd.

Het ‘bijsturen’ van een project

Binnen de IT-wereld wordt er veel beroep gedaan op elkaars kennis en ervaring. In overleg gaan met mijn collega’s om op zoek te gaan naar oplossingen of inzicht te krijgen in de juiste afbakening van het project is ‘een must’. Door te overleggen binnen het team en met de experten verwierf ik zelf ook meer inzicht in de aanpak van problemen. Deze zijn niet altijd aan je eigen inzet en vaardigheden gerelateerd maar meestal aan onverwachte basic-factoren van programma’s.
